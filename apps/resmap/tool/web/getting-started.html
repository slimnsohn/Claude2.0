Getting started

From a clean machine to a queryable cross-venue divergence dataset. Commands are PowerShell on Windows; the venv python is .venv\Scripts\python.exe (shown as python below). New to the concepts? Read how it works first.

0Prerequisites

  • PostgreSQL 16 — the system of record. (Windows: winget install PostgreSQL.PostgreSQL.16; full steps in _docs/SETUP_WINDOWS.md.)
  • Python 3.13.
  • Claude Code CLI (claude on PATH) — rule parsing runs through claude -p, so there's no API key or per-call cost.

1Create the database

Create the role + two databases (prod and a test DB) and load the schema into both:

$psql = "C:\Program Files\PostgreSQL\16\bin\psql.exe"
& $psql -U postgres -c "CREATE ROLE resmap LOGIN PASSWORD 'resmap';"
& $psql -U postgres -c "CREATE DATABASE resmap OWNER resmap;"
& $psql -U postgres -c "CREATE DATABASE resmap_test OWNER resmap;"
$env:PGCLIENTENCODING = "UTF8"   # schema has UTF-8 box chars
& $psql -v ON_ERROR_STOP=1 -f db\schema.sql "postgresql://resmap:resmap@localhost:5432/resmap"
& $psql -v ON_ERROR_STOP=1 -f db\schema.sql "postgresql://resmap:resmap@localhost:5432/resmap_test"
Windows psql gotcha: put flags (-f, -v) before the connection URL, or psql silently ignores them.

2Python environment & .env

python -m venv .venv
.\.venv\Scripts\python -m pip install -r requirements.txt
copy .env.example .env       # then edit if your DB creds differ
.\.venv\Scripts\python -m pytest tests\ -q     # sanity: everything green

.env holds DATABASE_URL, TEST_DATABASE_URL, and CLAUDE_CLI_MODEL (default sonnet). No venue keys are needed — all three venues ingest from public endpoints.

3Ingest markets

Pull every open market from all three venues. Settlement text is captured verbatim and hashed; re-running only writes a new snapshot when a venue actually edits the rules.

python -m ingest.run                       # all venues
python -m ingest.run --venue polymarket --max-pages 5   # a quick bounded run

Run it again — it's idempotent: unchanged markets write nothing. A genuine mid-market rule edit produces a rule_change_events row (a sellable alert on its own). Classify those edits cosmetic vs material with:

python -m parse.classify_changes

4Extract structured rules

Claude reads each market's verbatim rules and returns 10 structured fields (source, cutoff, tie handling, threshold, …). Highest-volume markets first; run in batches.

python -m parse.rule_parser --limit 20
python -m parse.rule_parser --limit 3 --dry-run    # preview without writing
Human review is the moat. Every parse lands reviewed = false. The LLM extraction is cheap and cloneable; a curated, verified settlement-semantics database is not. There is deliberately no auto-accept.

5Review parses & curate sources

python -m parse.review_cli list                 # unreviewed, shakiest first
python -m parse.review_cli show 7d45baf9        # parse beside the verbatim rules
python -m parse.review_cli approve 7d45baf9

Sources are normalized to short canonical authorities ("FIFA", "NASA GISS LOTI"). When two rows name the same authority differently, merge them so they stop looking like a divergence:

python -m parse.review_cli list-sources
python -m parse.review_cli merge-source "Fifa official body" "FIFA"
python -m scripts.cluster_sources               # let claude propose merges (you approve)

6Score cross-venue divergence

Find lookalike Polymarket↔Kalshi pairs and compare their parsed interpretations across four axes (source, cutoff, tie, threshold) → true_match / near_match / false_friend.

python -m parse.equivalence

For the full registry (slow to match), use the staged pipeline that caches candidates:

python -m scripts.equivalence_pipeline match --out pairs.json --top 12
python -m parse.rule_parser --ids ID1 ID2 ...   # parse any sides it flags as unparsed
python -m scripts.equivalence_pipeline compare --pairs pairs.json

7Serve the API

Mint a key (only its hash is stored), then start the read-only API:

python -m scripts.make_api_key --label "me" --rate 120    # prints the key once
python -m uvicorn tool.api.main:app --port 8077
EndpointReturns
GET /marketsmarkets (filter by venue/category/status)
GET /markets/{id}/rulescurrent parsed rules + snapshot ref
GET /equivalencesscored pairs (filter match_type / min_risk)
GET /rule-changesthe rule-change alert feed
curl -H "X-API-Key: YOUR_KEY" "http://127.0.0.1:8077/equivalences?match_type=false_friend"

8Open the dashboard

dashboard.html consumes the API: enter the API URL + your key and Load. Pairs are flagged true_match / near_match / false_friend with plain-language verdicts, alongside the rule-change feed. On Windows, start.bat launches the API and opens the dashboard in one step.

9Export for analysis

Snapshot the structured layers to partitioned Parquet (markets/parsed_rules/rule_changes by venue) — read it from DuckDB, pandas, or anything Arrow-aware:

python -m export.to_parquet --out ./export/parquet
-- from DuckDB:
SELECT * FROM read_parquet('export/parquet/equivalences/*.parquet')
WHERE match_type = 'false_friend';

10Keep it fresh automatically

A scheduled task runs scripts/daily_refresh.bat (ingest + export) daily. Parsing and equivalence stay manual — they use Claude and depend on human review.

schtasks /Create /TN "ResMap Daily Refresh" /TR "...\scripts\daily_refresh.bat" /SC DAILY /ST 06:00 /F

Command cheatsheet

python -m ingest.run                       # 3. pull markets + detect rule changes
python -m parse.classify_changes           #    label rule edits cosmetic/material
python -m parse.rule_parser --limit 20     # 4. extract rules (claude -p)
python -m parse.review_cli list            # 5. review queue
python -m parse.review_cli merge-source A B#    curate the source registry
python -m parse.equivalence                # 6. score divergence
python -m scripts.make_api_key --label x   # 7. mint an API key
python -m uvicorn tool.api.main:app        #    serve the API
python -m export.to_parquet                # 9. analytical snapshot
pytest tests\ -q                           #    run the test suite
